# public static void main(String[] args)

What does args mean?

我们可以通过命令行来编译 java 文件 -> javac xxx.java java xxx / java ./xxx

这两段命令分别是编译 compiler 和解释 interpreter

若在 java xxx 后面追加字符串如 abcd -> java xxx abcd

那么，这个 args String 数组的第 0 位将会是 abcd

至此，我们会更加熟悉一些 command 背后发生了什么

# Git: SHA-1

git 采用了 SHA-1 标识各个版本和各个文件，我们会在 git log 后见到这些哈希值

In cryptography, SHA-1 (Secure Hash Algorithm 1) is a hash function which takes an input and produces a 160-bit (20-byte) hash value known as a message digest – typically rendered as 40 hexadecimal digits. It was designed by the United States National Security Agency, and is a U.S. Federal Information Processing Standard.The algorithm has been cryptographically broken，but is still widely used.

在密码学中，SHA-1（安全哈希算法 1）是一种哈希函数，它接受输入并生成一个 160 位（20 字节）哈希值，称为消息摘要 —— 通常呈现为 40 个十六进制数字。它由美国国家安全局设计，是美国联邦信息处理标准。该算法已被加密破解 ，但仍被广泛使用。

# Git: merge conflict

这一小部分将会解释 merge conflict 是如何发生的。

如何单机实现 merge conflict :

  • 在提交几个版本后，使用 git checkout -b anotherBranch xxxhashcode xxxhashcode 也就是版本的哈希值

  • 随后就进入 anotherBranch 对目标文件做出修改（特意制造 merge conflict）

  • 回到 main/master 分支，使用 git merge anotherBranch 。这样就可以特意的实现一个 merge conflict

在这次讲座中，主要讨论了时间复杂度。在这之前，我们或许已经见过像O(N2)O(N^2)O(N2)​​这样的表达。这部分的内容将会继续讨论与之相关的内容。

# Θ: Big-Theta

借讲座中出现的一个例子: R(N)=40sin(N)+4N2R(N) = 40 sin(N) + 4N^2R(N)=40sin(N)+4N2

其对应的 Θ 会是 Θ(N2)Θ(N^2)Θ(N2)

在这里我们作一个设置 N2=f(N)N^2 = f(N)N2=f(N)

故，以上的 Θ(N2)Θ(N^2)Θ(N2) 可以表示为Θ(f(N))Θ(f(N))Θ(f(N)) ，这是一个通用的表示。为了理解下面的内容，我们需要这样的表示。

至此，会发现这和 Big O Notation 的使用是一模一样的。但是其意义是不同的。

Θ，用这个代表 order of growth (o.o.g) 增长量级

Using “Big-Theta” instead of “order of growth” does not change the way we analyze code at all.

对于如上的R(N)R(N)R(N) 和 Θ(N2)Θ(N^2)Θ(N2) 的关系会有如下表示：

R(N)=40sin(N)+4N2∈Θ(N2)R(N) = 40 sin(N) + 4N^2 ∈Θ(N^2)R(N)=40sin(N)+4N2∈Θ(N2)

为什么会是∈呢？

Θ 表示的是 o.o.g ， 那么Θ(N2)Θ(N^2)Θ(N2) 表示的是抛物线N2N^2N2 的增长量级。那么像2N2+N2N^2+N2N2+N 也会对应Θ(N2)Θ(N^2)Θ(N2)。因此 Θ(N2)Θ(N^2)Θ(N2) 代表了一类的 o.o.g ，在这里R(N)R(N)R(N)​​属于这一类。

为什么要有这样的表达？

在继续讨论这个问题之前，先将式子通用化以便进行广泛的讨论

R(N)∈Θ(f(N))R(N) ∈ Θ(f(N))R(N)∈Θ(f(N)) ，R(N)R(N)R(N) 可以是任意函数

紧接着将伴随这样的不等式

k1⋅f(N)<=R(N)<=k2⋅f(N)k_1 · f(N) <= R(N) <= k_2 · f(N)k1​⋅f(N)<=R(N)<=k2​⋅f(N)，k1k_1k1​ k2k_2k2​ 均为正数常量

为了解释上面那个问题，先来说明这个不等式意味着什么

k1,k2k_1,k_2k1​,k2​​会是什么？

依旧以最开始的式子为例子

R(N)=40sin(N)+4N2R(N) = 40 sin(N) + 4N^2R(N)=40sin(N)+4N2 经过初步简化后会得到4N24N^24N2

那么，可以大致地取k1=3k_1 = 3k1​=3, k2=5k_2 = 5k2​=5​

代入到刚才的不等式，将会是

3N2<=40sin(N)+4N2<=5N23 N^2 <= 40 sin(N) + 4N^2 <= 5 N^23N2<=40sin(N)+4N2<=5N2​

将这个不等式用坐标系描绘，得到

from this link

可以看到，5 和 3 把R(N)R(N)R(N) 夹在它们之间了。所以这就是为什么可以取 5 和 3 -> 上下渐进原来的函数R(N)R(N)R(N)​ -> 这自然会扩展到极限的范畴

那么为了区别 O，这里特意指出 Θ 的一点：借k1,k2k_1, k_2k1​,k2​ 上下渐进原函数

# O: Big-Oh

O 实际上就是忽略了 Θ 的下界

R(N)∈O(f(N))R(N) ∈ O(f(N))R(N)∈O(f(N))

R(N)<=k2⋅f(N)R(N) <= k_2 · f(N)R(N)<=k2​⋅f(N)

O 常用的原因就在于，它标识了上界，也就是最坏的情况下的复杂度。大家通常会考虑最坏的情况

# Ω: Big-Omega

与 O 相反，忽略的 Θ 的上界，标识了下界。对应了最好的情况。

# Big-heta vs. Big-Oh

Informal meaning Family Family Members
Big-Theta Θ(f(N))Θ(f(N))Θ(f(N)) Order of growth is f(N). Θ(N2)Θ(N^2)Θ(N2) N2/2N^2/2N2/2, 2N22N^22N2, N2+38N+NN^2 + 38N + NN2+38N+N
Big-Oh O(f(N))O(f(N))O(f(N)) Order of growth is less than or equal to f(N). O(N2)O(N^2)O(N2) N2/2N^2/2N2/2, 2N22N^22N2, lg(N)lg(N)lg(N)
